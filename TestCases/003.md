# Test Case 002 — Login with Empty Username and Password

## **Test ID**
TC-003

## **Title**
Login text length

## **Description**
Verify that username and password fields accept text within the allowed length range.

## **Preconditions**
- Android emulator or BrowserStack device is running  
- Appium server is running and connected  
- Android Native Demo App is installed on the device  

## **Test Steps**
1. Navigate to the login screen.
2. Enter a randomly generated username with 8–50 characters + "gmail.com".
3. Enter a randomly generated password with 8–50 characters.
4. Click the “Login” button.

## **Expected Results**
1. Login screen is displayed.
2. User is able to proceed if input is within the allowed range.

## **Linked Automation**
- **Test:** `LoginScreen.test.js` 
- **Page:** `login.page.js`
